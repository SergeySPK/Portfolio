Проект 01 / 03
ИС для оценивания преподавателей
50тестов pytest
87%покрытие кода
4HTML-шаблона
Задача клиента
Колледж проводил оценку преподавателей через бумажные анкеты. Обработка одного потока занимала 2–3 рабочих дня, данные терялись, сравнение за разные семестры было невозможным. Нужна была цифровая система, работающая с любого устройства через браузер.
Мои действия
Спроектировал MVC-архитектуру на Flask с разделением на app.py и database.py. Реализовал динамическую схему БД: критерии оценивания добавляются через CSV без перезапуска сервера. Написал адаптивный UI с Jinja2 и Fetch API для сохранения оценок без перезагрузки страницы. Покрыл бизнес-логику 50 unit-тестами с pytest, настроил деплой на Render.com.
Результат
Время обработки результатов анкетирования сократилось с 2–3 дней до нескольких секунд. Администратор получил панель с сортировкой, фильтрацией и управлением столбцами. Система задеплоена и доступна 24/7 через HTTPS.
Python Flask SQLite Jinja2 pytest Render.com
💡 После проекта: важно выносить бизнес-логику в отдельный модуль с первого дня — это делает тестирование в 10 раз проще и избавляет от «спагетти-кода» по мере роста функционала.

Проект 02 / 03
Парсер и нормализатор адресов
130+улиц в базе
NLPбиграммный поиск
2таблицы SQLite
Задача клиента
Организации поступали адреса в свободной форме: с опечатками, пропущенными словами, смешением регистров. Нужно было автоматически разобрать адрес на компоненты (город, посёлок, улица, дом, квартира) и записать исправленные данные в БД.
Мои действия
Реализовал трёхфазовый NLP-алгоритм на основе биграммного сравнения (коэффициент Жаккара). Система собирает кандидатов для каждого компонента, затем выбирает лучшего по приоритету (группа слов vs одиночное слово). Логика извлечения номера дома/квартиры обрабатывает сложные случаи: «д.10/2, кв 45», «ул. Пушкина, 8 стр.1» и т.д. Веб-интерфейс на Flask с просмотром таблиц T1 и T2.
Результат
Система корректно разбирает адреса с грамматическими ошибками и нестандартными сокращениями. Точность определения улицы на тестовой выборке из 50 адресов — около 90%. Данные сохраняются в нормализованном виде в SQLite с поддержкой каскадного удаления.
Python Flask SQLite NLP (биграммы) regex flask-cors
💡 После проекта: нечёткий поиск — нетривиальная задача. Метрика Жаккара отлично работает для коротких строк, но требует тонкой настройки порогов для каждой категории данных.

Проект 03 / 03
Детектор лиц (фото / видео / веб-камера)
MTCNNнейросеть
3режима работы
160pxлицо в кропе
Задача клиента
Нужен инструмент для автоматического извлечения лиц из фотоархивов и видеозаписей. Ручная обработка больших объёмов видео занимала часы работы. Требовался простой GUI, работающий в том числе с веб-камерой в реальном времени.
Мои действия
Интегрировал нейросеть MTCNN (Multi-task Cascaded Convolutional Networks) для детекции лиц. Реализовал три режима: изображение, видеофайл (кадр за кадром с таймстампом), веб-камера (real-time с обновлением через Tkinter Canvas). Для каждого обнаруженного лица создаётся кроп 160×160px с именем файла по уверенности (confidence). GUI на tkinter с радиокнопками, вводом пути и кнопками управления камерой.
Результат
Программа обрабатывает фотографию за 1–3 секунды и видео — в зависимости от длины. Все найденные лица сохраняются в структурированные папки с именем исходного файла. Уверенность детекции от 0.90 и выше даёт качественные кропы без ложных срабатываний.
Python OpenCV MTCNN TensorFlow Pillow tkinter
💡 После проекта: работа в реальном времени требует баланса между точностью и скоростью. Уменьшение входного кадра ускоряет детекцию, но снижает точность для мелких лиц.
